Byte Pair Encoding (BPE)

Start from characters + end-of-word marker </w>. It distinguishes word-final vs. word-internal positions (e.g., "er" in "hugger" vs. "error") and enables correct detokenization. Iteratively merge the most frequent adjacent pair.

\mathrm{pair}^* = \arg\max_{(a,b)} \;\mathrm{freq}(a,b)

Inference: apply learned merges in order. A word is split into characters, then each merge rule is applied left-to-right. The order matters — earlier merges get priority.

Byte Pair Encoding (BPE)

Начинаем с символов + маркер конца слова </w>. Он отличает конечные позиции от внутренних (напр., "er" в "hugger" vs. "error") и обеспечивает корректную детокенизацию. Итеративно объединяем самую частую соседнюю пару.

\mathrm{pair}^* = \arg\max_{(a,b)} \;\mathrm{freq}(a,b)

Инференс: применяем выученные слияния по порядку. Слово разбивается на символы, затем каждое правило слияния применяется слева направо. Порядок важен — ранние слияния приоритетнее.

Byte Pair Encoding (BPE)

Хәрефләрдән + сүз ахыры билгесе </w> башлыйбыз. Ул сүз ахырын эчке позициядән аера (мәс., "er": "hugger" vs. "error") һәм дөрес детокенизацияне тәэмин итә. Иң еш очраган күрше парны итератив берләштерәбез.

\mathrm{pair}^* = \arg\max_{(a,b)} \;\mathrm{freq}(a,b)

Инференс: өйрәнелгән берләшүләрне тәртип буенча кулланабыз. Сүз хәрефләргә бүленә, аннары һәр берләшү кагыйдәсе сулдан уңга кулланыла.

WordPiece

Like BPE, but merges maximize pointwise mutual information (PMI): favors rare tokens that co-occur often. The ## prefix marks continuations — same role as </w> but from the other side: it tells the model where a word does not begin.

\mathrm{score}(a,b) = \frac{\mathrm{freq}(ab)}{\mathrm{freq}(a) \cdot \mathrm{freq}(b)}

Inference: greedy longest-match from vocab. Start at the beginning of the word, find the longest token in vocab, consume it, repeat. Unknown chars become [UNK].

WordPiece

Как BPE, но слияния максимизируют поточечную взаимную информацию (PMI): предпочтение редким, но часто соседствующим токенам. Префикс ## — та же роль, что </w>, но с другой стороны: он показывает, где слово не начинается.

\mathrm{score}(a,b) = \frac{\mathrm{freq}(ab)}{\mathrm{freq}(a) \cdot \mathrm{freq}(b)}

Инференс: жадный поиск самого длинного совпадения из словаря. Начинаем с начала слова, находим самый длинный токен в словаре, потребляем, повторяем. Неизвестные символы → [UNK].

WordPiece

BPE кебек, ләкин берләшүләр ноктавый үзара мәгълүматны (PMI) максимальләштерә: сирәк, ләкин еш бергә очраган токеннарга өстенлек. ## префиксы — </w> белән бер үк роль, ләкин каршы яктан: сүз кайда башланмаганын күрсәтә.

\mathrm{score}(a,b) = \frac{\mathrm{freq}(ab)}{\mathrm{freq}(a) \cdot \mathrm{freq}(b)}

Инференс: сүзлектән иң озын тапкычлык. Сүз башыннан башлап, сүзлектәге иң озын токенны табабыз, кулланабыз, кабатлыйбыз. Билгесез символлар → [UNK].

Unigram Language Model

Opposite of BPE: start with a huge vocab (all substrings up to length 5), iteratively prune pieces whose removal hurts corpus likelihood least. Segmentation via Viterbi — dynamic programming that finds the highest-probability left-to-right path through all possible splits.

x^* = \arg\max_{x \in S(w)} \sum_{i=1}^{|x|} \log P(x_i)

Note: simplified version (real SentencePiece uses EM for P(x_i)). Inference: Viterbi on final vocab — same DP as during training.

Unigram Language Model

Противоположность BPE: начинаем с огромного словаря (все подстроки длиной до 5), итеративно отсекаем элементы, чьё удаление меньше всего вредит правдоподобию корпуса. Сегментация по Витерби — динамическое программирование, находящее путь максимальной вероятности слева направо.

x^* = \arg\max_{x \in S(w)} \sum_{i=1}^{|x|} \log P(x_i)

Примечание: упрощённая версия (настоящий SentencePiece использует EM для P(x_i)). Инференс: Витерби на финальном словаре — та же DP что при обучении.

Unigram Language Model

BPE га каршы: зур сүзлектән (озынлыгы 5 кә кадәр барлык подстрокалар) башлап, юкка чыгаруы иң аз зыян китерүче кисәкләрне кисеп ташлыйбыз. Сегментация Витерби аша — сулдан уңга иң югары ихтималлыклы юлны табучы динамик программалаштыру.

x^* = \arg\max_{x \in S(w)} \sum_{i=1}^{|x|} \log P(x_i)

Искәрмә: гадиләштерелгән версия (чын SentencePiece P(x_i) өчен EM куллана). Инференс: финаль сүзлектә Витерби — өйрәтүдәге DP белән бер үк.

Byte-Level BPE

Same BPE on raw UTF-8 bytes. Base vocab = 256 (all possible byte values). Any language/script, no pre-tokenization needed. Hover over hex tokens like <c3> to see the decoded character.

|\mathcal{V}_0| = 256 \quad(\text{all possible byte values})

Trade-off: universality at the cost of longer sequences for non-ASCII. café → 5 bytes; 猫 → 3 bytes. Inference: same as BPE — apply merges in order, but on bytes.

Byte-Level BPE

Тот же BPE на сырых байтах UTF-8. Базовый словарь = 256 (все возможные значения байт). Любой язык, без предобработки. Наведите на hex-токены вроде <c3> чтобы увидеть декодированный символ.

|\mathcal{V}_0| = 256 \quad(\text{все возможные значения байт})

Компромисс: универсальность за счёт длинных последовательностей для не-ASCII. café → 5 байт; 猫 → 3 байта. Инференс: как BPE — слияния по порядку, но на байтах.

Byte-Level BPE

Шул ук BPE чимал UTF-8 байтларында. Нигез сүзлек = 256 (барлык мөмкин байт кыйммәтләре). Теләсә нинди тел, алдан эшкәртүсез. Hex-токеннарга (<c3>) күрсәтеп декодланган символны күрегез.

|\mathcal{V}_0| = 256 \quad(\text{барлык мөмкин байт кыйммәтләре})

Бәя: универсальлык ASCII-дан тыш озынрак эзлекле бирә. café → 5 байт; 猫 → 3 байт. Инференс: BPE кебек — берләшүләрне тәртип буенча, ләкин байтларда.

Comparison

All 4 tokenizers trained on the same corpus, tokenizing the same input. Green = fewest tokens (most efficient), amber = most tokens.

\text{BPE} \quad \text{WordPiece} \quad \text{Unigram} \quad \text{Byte BPE}

Key difference: BPE uses frequency, WordPiece uses PMI, Unigram uses probabilistic pruning, Byte BPE operates on raw bytes.

Сравнение

Все 4 токенизатора обучены на одном корпусе, токенизируют один вход. Зелёный = меньше токенов (эффективнее), янтарный = больше токенов.

\text{BPE} \quad \text{WordPiece} \quad \text{Unigram} \quad \text{Byte BPE}

Ключевое отличие: BPE использует частоту, WordPiece — PMI, Unigram — вероятностное отсечение, Byte BPE работает на сырых байтах.

Чагыштыру

Барлык 4 токенизатор бер корпуста өйрәтелгән, бер кертүне токенлаштыра. Яшел = аз токен (эффективрак), янтарь = күбрәк токен.

\text{BPE} \quad \text{WordPiece} \quad \text{Unigram} \quad \text{Byte BPE}

Төп аерма: BPE ешлык куллана, WordPiece — PMI, Unigram — ихтималлык белән кисү, Byte BPE чимал байтларда эшли.
